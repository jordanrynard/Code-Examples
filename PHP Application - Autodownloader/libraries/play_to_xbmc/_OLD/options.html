Update 1.5.0.0:

  • Added context menus to show images on XBMC (Beta). This feature on XBMC is a little unstable and images like Gif is not supported.
  • Added context menus to play and queue direct links to music files. Supported extensions are: 'mp3', 'ogg', 'midi', 'wav', 'aiff', 'aac', 'flac', 'ape' and 'wma'.
  • Added support for MyCloudPlayers.com (Many thanks to cuckoo for the code work).
  • Added a donation page to the option tabs. If you like my work, consider buying me a coffee so I could spend more hours awake working on Play to XBMC.

Update 1.4.3.0:

  • Added support for XBMCtorrent. You can now use the context menus to play magnet links. For more information on XBMCtorrent, click HERE.

Update 1.4.2.1:

  • Fixed a bug where Vimeo urls that ends with a # would be detected as invalid.
  • Allow context menus on Google redirected YouTube URLs.

Update 1.4.2.0:

  • Fixed a bug with Twitch.tv where certain URL patterns are not correctly validated and therefore, leaving the 'Play' button disabled.
  • Added ability to queue entire Soundcloud sets as a playlist in XBMC.
  • Added support to play Khan Academy videos.
  • Added support for Hulu videos, with a catch. You must install the unofficial Hulu plugin by BlueCop. Find out more by clicking HERE. (Many thanks to Yulius for this)

Update 1.4.1.0:

  • Show a tooltip to inform the user that the extension has been updated.
  • Changelist now available in the options menu. If you haven't seen this list before, I recommend that you go through this list once. You might find a feature that you've been waiting for.
  • Added support for Twitch.tv, but there's a catch: Only live streams supported. I can't seem to find a way to play non-live videos with the Twitch.TV plugin on XBMC. Will add it in once I have an idea.

Update 1.4.0.0:

  • Support for context menus on YouTube links have now been added. This is probably the second most requested feature after YouTube playlists. Context menus for other supported sites are coming, I just can't wait to bring you this update first :)
  • Fixed a bug where the buttons do not activate for Vimeo videos under a group.

Update 1.3.4.2:

  • -Added support for the other playlist page. (Thanks to Mark Cunningham for codework)

Update 1.3.4.0:

  • -Added support for YouTube playlists (Many thanks to Shlomi for doing the codework). Probably one of the most requested features, click on the dropdown menu to Queue entire YouTube playlists.
  • -Added support for key bindings (Arrow keys, Backspace, Enter, I and C). With the extension popup open, you can use the listed keys to navigate XBMC.

Update 1.3.3.0:

  • -Added support for LiveLeak videos

Update 1.3.2.0:

  • -Added support for MixCloud* (Works with SoundCloud in the same playlist. Read more below)

Update 1.3.1.0

  • -Added support for multiple XBMC hosts and easy switching

Update 1.3.0.1:

  • -Minor bug fixes

Update 1.3.0.0:

  • -Added support for eBaumsWorld and SoundCloud* (SoundCloud works in a different playlist, more details below)
  • -Added the ability to set the Repeat Mode on XBMC. Enable this in the settings.
  • -Added the ability to play something next
  • -Added the ability to remove the currently playing item from the playlist
  • -Show the total number of items in the current playlist and the position it is currently in.

* Because XBMC handles audio and video content with different players, they also go into different playlists. That also means that you can't mix audio (MixCloud/SoundCloud) and video (YouTube etc) contents in the same queue.

Update 1.2.6.1:

  • -Added support for DailyMotion. Again, this requires the DailyMotion plugin to be installed on XBMC
  • -Fixed the 'Skip' and 'Previous' not working on newer versions of XBMC (v12+)
  • -Last time I tested, I couldn't get a single CollegeHumor video to play, this is an issue with the plugin on XBMC, which is no longer supported.
